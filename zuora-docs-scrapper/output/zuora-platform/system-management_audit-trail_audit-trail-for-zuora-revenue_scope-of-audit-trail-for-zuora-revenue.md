# Scope of Audit Trail for Zuora Revenue

This topic explains the scope and details of the audit trail functionality within Zuora Revenue.

## Audit trail scope

-   Track, log, and report on template changes. You can audit the changes on the following settings:

    -   Cost

    -   Forecast

    -   Mass action

    -   Period open/close

    -   Performance obligation

    -   Revenue contract batch

    -   Revenue contract template

    -   SSP

    -   Transfer accounting

-   Track, log, and report on a user's login history.
-   Track, log, and report on setting changes. You can audit the changes on the following settings:
    -   Account setups
    -   Accounting setups
    -   Approvals
    -   Bundle configuration
    -   Contract modification
    -   Currency
    -   Data augmentation or stage validation
    -   Disclosure aggregation
    -   Email
    -   Events
    -   Financing
    -   Functions
    -   Holds
    -   Integration mapping
    -   Interface
    -   Labels
    -   Lookups
    -   Organization
    -   Period open/close task
    -   Periods
    -   Pre/post processor
    -   Profiles
    -   Reports
    -   Revenue books
    -   Role organization
    -   Role privilege
    -   Roles
    -   Segments
    -   Soft freeze
    -   Upload
    -   Variable consideration
-   Track, log, and report on template changes. You can audit the changes on the following settings:
    -   Cost
    -   Forecast
    -   Mass action
    -   Period open/close
    -   Performance obligation
    -   Revenue contract batch
    -   Revenue contract template
    -   SSP
    -   Transfer accounting
