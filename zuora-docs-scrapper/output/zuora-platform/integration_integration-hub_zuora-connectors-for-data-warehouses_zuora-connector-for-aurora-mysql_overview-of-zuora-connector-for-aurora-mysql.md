# Overview of Zuora Connector for Aurora MySQL

Access Zuora data directly in Aurora MySQL with daily refresh for seamless, insight-driven decisions

Note:

MySQL Version Support Disclaimer

Effective September 2025:

-   Standard support for MySQL 5.7 is ending.

-   Minimum supported version: MySQL 8.0 (Aurora v3).

-   The MySQL connector will continue to run on 5.7, but treated as best-effort only. New features and enhancements are not guaranteed to work. Issues on 5.7 will receive best-effort support, but fixes may not include backward compatibility.


Related Cloud EOL Notices:

MySQL ([MySQL 5.7 EOL](https://www.mysql.com/support/eol-notice.html), [AWS Aurora v2 / MySQL 5.7 EOL](https://docs.aws.amazon.com/AmazonRDS/latest/AuroraUserGuide/Aurora.MySQL57.EOL.html), [Azure MySQL version policy](https://learn.microsoft.com/en-us/azure/mysql/concepts-version-policy))

Enable your organization to access Zuora data directly through Aurora MySQL. This integration simplifies the setup process, eliminating common technical hurdles related to data extracts, integrations, and schema updates.

Once configured, your data is automatically refreshed daily within Aurora MySQL, granting your team immediate access to extensive subscription operational and analytical data from Zuora Billing. This seamless access enables your teams to swiftly make informed decisions based on valuable insights.

## How the Zuora Connector for Aurora MySQL works

The Zuora Connector for Aurora MySQL facilitates a direct connection between your Zuora account and your Aurora MySQL environment. This connection is established through a one-time setup process during which you configure your Aurora MySQL credentials.

Once configured, the connector automates the synchronization of your Zuora data into your specified Aurora MySQL environment on a daily basis. Tailored to seamlessly integrate with Aurora MySQL, this connector enhances your data storage and analytics capabilities within your infrastructure.
