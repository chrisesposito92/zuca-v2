# Allow access

Allow access

If your Postgres database is protected by security groups or other firewall settings, you will need to have the data-syncing service's static IP available to complete Step 1.

Allow write access to a portion of your RDS or Aurora PostgreSQL database. This step includes two procedures.

1.  [Configure the security group](/zuora-platform/integration/integration-hub/zuora-connectors-for-data-warehouses/zuora-connector-for-aws-aurora-postgres/set-up-zuora-connector-for-aws-aurora-postgres/allow-access/configure-the-security-group)
2.  [Configure network ACLs](/zuora-platform/integration/integration-hub/zuora-connectors-for-data-warehouses/zuora-connector-for-aws-aurora-postgres/set-up-zuora-connector-for-aws-aurora-postgres/allow-access/configure-network-acls)
