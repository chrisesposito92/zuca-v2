# Setup SSO with Okta using SAML in OneID

This guide explains how to set up single sign-on (SSO) with Okta using SAML in OneID, including steps to add the Zuora OneID application, obtain metadata, assign applications, and enable provisioning.

This article describes how to configure Okta to enable single sign-on with OneID. For more information about SSO in OneID, see Configure single sign-on for OneID.

## Procedure

1.  [Add Zuora OneID application to OKTA](/zuora-revenue/advanced-revenue-operations/configure-single-sign-on-for-oneid/setup-sso-with-okta-using-saml-in-oneid/add-zuora-oneid-application-to-okta)
2.  [Obtain OKTA SAML metada URL](/zuora-revenue/advanced-revenue-operations/configure-single-sign-on-for-oneid/setup-sso-with-okta-using-saml-in-oneid/obtain-okta-saml-metadata-url)
3.  [Assign the Zuora OneID application to OKTA users](/zuora-revenue/advanced-revenue-operations/configure-single-sign-on-for-oneid/setup-sso-with-okta-using-saml-in-oneid/assign-the-zuora-oneid-application-to-okta-users)
4.  [Enable provisioning using SCIM API](/zuora-revenue/advanced-revenue-operations/configure-single-sign-on-for-oneid/setup-sso-with-okta-using-saml-in-oneid/enable-provisioning-using-scim-api)
