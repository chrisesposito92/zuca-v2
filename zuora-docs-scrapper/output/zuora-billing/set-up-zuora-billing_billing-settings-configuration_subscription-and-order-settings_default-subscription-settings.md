# Default subscription settings

This topic explains the default subscription settings such as term type, renewal options, and pricing attributes to manage subscription lifecycle effectively.

Create the usage charge with the following pricing attributes:

| Settings | Description |
| --- | --- |
| Default Term Type | Select Termed or Evergreen. The Default Term Type setting specifies whether a new subscription has a set term by default or it does not expire.Evergreen subscriptions have no end date, which means your customer will be automatically billed until the subscription is canceled.A termed subscription expires after the number of months specified in the initial term, and can be set to renew for additional renewal terms. |
| Default Initial Term (months) | Set the number of months to use for initial subscription terms. You can override this value when creating a subscription. |
| Default Renewal Term (months) | Set the number of months your subscription will renew for after the initial term expires. You can override this value when creating a subscription. |
| Default Subscriptions to Auto-Renew? | Specify whether subscriptions are to automatically renew after the initial term expires. See Renewing Subscriptions for more information.Yes: Your subscription will automatically renew for the number of months specified in the renewal term.No: The subscription expires after the initial term ends. |
| Require Customer Acceptance of Orders? | Specify whether you must enter a customer acceptance date to activate a subscription. The customer acceptance date is the date on which the customer accepts the services or products within a subscription. See Activating a Subscription for more information. |
| Require Service Activation of Orders? | Specify whether you must enter a service activation date to activate a subscription. The service activation date is the date in which the customer is provided access to the services or products within a subscription. |
| Enable Subscriptions to be Invoiced Separately? | Zuora Billing can separate a single charge and bill that charge independently to give you the flexibility to invoice charges independently, allowing you to improve your cash flow by immediately invoicing a charge.See Invoicing Subscriptions Separately for more information. |
| Enable Automatic Price Change when Subscriptions are Renewed? | Select Yes to apply an automated price change when subscriptions are renewed. Typically, this change is a price increase, and is referred to as an "Uplift." When you set enable this, the Default Price Change for Product Charges and Default Price Increase Percentage Value fields appear. See Automatic Price Change (Uplift) for Subscriptions for more information. |
| Enable Feature Specification in Products and Subscriptions? | When the Entitlements feature is enabled on your tenant, you can set the Enable Feature Specification in Product and Subscriptions default value to enable or disable the features functionality. If set to Yes, you can create and update features in product catalog, and you can add features in products and subscriptions. |
| Default Price Change for Product Charges | Specify the default behavior of price changes for products after a subscription renewal.No Change does not allow price changes for products.Percentage Increase lets you choose a percentage value for the price increase. Enter the percentage price change that you would like to apply in Default Price Increase Percentage Value.Use Latest Product Catalog Pricing uses the price for the product that is in the product catalog when a subscription is renewed. |
| Enable Subscription auto-renew when there's no product in subscription? | Specify whether to allow subscriptions that no longer have active product charges to auto-renew.Yes: allows subscriptions to follow your current subscription renewal settings.No: prevents auto-renewal on subscriptions that do not have any active product charges. |
| Subscription Auto Renew Job Type | Specify whether to trigger the auto-renew job based on the system default time or your specified time.System Default: Zuora triggers the auto-renew job using the system default time. The system default time is at 01:00 in the Zuora system-wide time zone. The Zuora US data center uses Pacific Time (PT). The Zuora EU data center uses Coordinated Universal Time (UTC).Tenant Specified Time: Zuora triggers the auto-renew job based on the time you set in the Subscription Auto Renew Job Time setting. |
| Subscription Auto Renew Job Time | Specify the run time when auto-renew is triggered. This setting is only available if you set the Subscription Auto Renew Job Type setting to Tenant Specified Time.The run time you set is using your tenant time zone. |
| Calculate Tax for TCB and ELP Metrics? | Specify whether to generate tax for the TCB and ELP metrics when an Order is created. This is for both new order creation and the initial data migration.Yes: Zuora will call the tax engine to calculate estimated taxes for Order Metrics. This will populate the Tax field on the ELP and TCB metrics. If the tenant is using an external tax vendor, this will incur in tax API calls during the Order creation process.No: the default value. Zuora will not call tax engine during the Order creation. The Tax field on the ELP and TCB metrics will be NULL |
| Update Rate Plan Charge Trigger Condition | Specify whether to enable direct update of the subscription rate plan charge trigger condition. The charge must not have been invoiced.YesYes:You will see an option to edit the Trigger Condition field for each charge in the Product & Charges area of the subscription details page. You can update the trigger condition for a charge through the edit link. See Updating the Trigger Condition for a Subscription Rate Plan Charge. Note that you can only edit the trigger condition if the charge is not invoiced.Alternatively, you can also use the Update action to update the trigger condition for a charge through setting the TriggerEvent field. See Updating the Trigger Condition for a Subscription Rate Plan Charge.No (Default):You will not see an option to edit the Trigger Condition field for each charge in the Product & Charges area of the subscription details page.Also, you cannot update the trigger condition for a charge through the Update action. |
| Allow Update Subscription Trigger Dates? | Specify whether to allow direct update of the subscription trigger dates. The subscription must be on Version 1, which means it has not been amended before.Yes:You will see the set activation dates button on the subscription details page. You can update the trigger dates for a subscription through the button. See Updating the Trigger Dates for a Subscription. Note you can only update the trigger dates if the subscription is on Version 1.Alternatively, you can also use the Update action to update the trigger dates for a subscription through setting the ContractEffectiveDate, ServiceActivationDate, or ContractAcceptanceDate field. See Updating the Trigger Dates for a Subscription.No (Default):You will not see the set activation dates button to update the trigger dates for a subscription on the subscription details page.Also, you cannot update the trigger dates for a subscription through the Update action. |
| Enable Account Level Discount? | Specify whether to allow applying account level discounts to your subscriptions.Yes: You can apply account level discount charges to your subscriptions by creating a new subscription, adding a product, or updating an existing discount charge through both Zuora UI and API.No: Account level discounts are not enabled. |
| Enable Force Deletion for Account? | Specify whether to allow force deletion for an account under the specific condition as described below:Yes: You can force delete an account that is the subscription owner while the invoice owner is a different account. Force deleting this account deletes all its subscriptions, but the relevant invoices will not be impacted.No: You cannot delete an account that is the subscription owner of a subscription while the invoice owner is a different account. |
| Skip Amendment Generation by Subscribe? | When creating a subscription using the Subscribe and Amend UI or API, specify whether to skip generating a Composite type amendment and an equivalent order. This setting is available only if Orders Harmonization is enabled on your tenant.Yes: Skip generating a Composite type amendment and an equivalent order when creating a subscription using the Subscribe and Amend UI or API on your Orders Harmonization tenant. Yes is the default value on the tenants with Orders Harmonization enabled as of or after Zuora Release 2022.09.R2.No: Generate a Composite type amendment and an equivalent order when creating a subscription using the Subscribe and Amend UI or API on your Orders Harmonization tenant. No is the default value on the tenants with Orders Harmonization enabled before Zuora Release 2022.09.R2. |
